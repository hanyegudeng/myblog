创建私有pods库

首先创建一个自己的git仓库。我创建的仓库是：http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git

1.pod repo add USeOnLineLib http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git
2.pod lib create USeOnLineLib
如果死在这里：Setting up CocoaPods master repo，退出，进到Example,update下pods就可以了。（出现这个原因是本地的pods被删除了）
3.推送创建的文件到git上

git add .
git commit -s -m "Initial Commit of Library"
git remote add origin http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git
git push origin master

4.因为pods必须要有tag所以创建一个tag

git tag -m "first release" 0.1.0
git push --tags

5.修改podspec文件。可以直接打开Example下的项目

  s.name             = 'USeOnLineLib'
  s.version          = '0.1.0'
  s.summary          = 'USeOnLineLib'

  s.description      = <<-DESC
TODO: Add long description of the pod here.
                       DESC

  s.homepage         = 'http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git'
  s.license          = { :type => 'MIT', :file => 'LICENSE' }
  s.author           = { 'Yun' => 'xxxxxx@163.com' }
  s.source           = { :git => 'http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git', :tag => '0.1.0' }

  s.ios.deployment_target = '8.0'

  s.source_files = 'USeOnLineLib/Classes/**/*'
  
  # s.resource_bundles = {
  #   'USeOnLineLib' => ['USeOnLineLib/Assets/*.png']
  # }

  # s.public_header_files = 'Pod/Classes/**/*.h'
  # s.frameworks = 'UIKit', 'MapKit'
  # s.dependency 'AFNetworking', '~> 2.3'
end

s.dependency 'AFNetworking', '~> 2.3' 这样就附件了AFNetworking库，多个就多行

6.验证podspec文件是否正确pod lib lint
如果出现下面信息，就说明验证成功


USeOnLineLib passed validation.

7.提交podspec（如果更新pod后，也可以使用该命令重新发布提交）

pod repo push USeOnLineLib USeOnLineLib.podspec

8.在项目中使用该私有pods。前提是使用者必须有该库的访问权限

pod 'USeOnLineLib',:git=>"http://xxx.xxx.xx.x:28080/Yun/USeOnLineLib.git"

在pods里面使用别的pods的类的时候导入的时候用import <>避免出问题

如何是用Targets来管理开发测试包

我们在开发APP的过程中难免会遇到开发测试分离的情况，最常见的就是API地址，是不同的，这个可以通过宏定义或者注释之类的来加以解决，但是打包的时候万一忘了就完蛋喽。其实xcode就能帮我们防呆，那就是创建多个Target来管理不同的环境，配置不同的plist以及对应的程序图标版本号等等的信息。具体做法如下：

1.选择原有的target，复制一份，名字可以修改，也可以不管
2.在Build Settings里面的Preprocessor Macros添加不同的DEVELOPMENT=0
3.在用到API host的地方加入判断

4.如果是用swift编写项目，需要在Swift Compiler - Custom Flags里面把DEVELOPMENT添加进去才可以起效
```

if DEVELOPMENT

define SERVER_URL @"http://dev.server.com/api/"

define API_TOKEN @"DI2023409jf90ew"

else

define SERVER_URL @"http://prod.server.com/api/"

define API_TOKEN @"71a629j0f090232"

endif


4.如果程序中使用cocoapods，并且使用了workspace，就要稍微配置下podfile文件了。

abstract_target 'targetBlock' do

platform :ios, "7.0"

xcodeproj 'xx/xx.xcodeproj'

pod 'AFNetworking', '~> 3.0'

target 'testing target'
target 'product target'

end
```

5.如果后续添加文件什么的，记得选择两个target哦。要不另外一个target就没有对应的文件喽

6.注意：
swift需要在Other Swift Flags里面添加
"-D" "DEVELOPMENT"

AFNetworking添加Cookie以及Header

添加Cookie

NSMutableDictionary *cookieProperties = [NSMutableDictionary dictionary];
[cookieProperties setObject:@"TOKEN" forKey:NSHTTPCookieName];
[cookieProperties setObject:uuid forKey:NSHTTPCookieValue];
[cookieProperties setObject:@"xxx.xxx.xxx.xxx" forKey:NSHTTPCookieDomain];
[cookieProperties setObject:@"xxx.xxx.xxx.xxx" forKey:NSHTTPCookieOriginURL];
[cookieProperties setObject:@"/" forKey:NSHTTPCookiePath];
[cookieProperties setObject:@"0" forKey:NSHTTPCookieVersion];

NSHTTPCookie *cookie = [NSHTTPCookie cookieWithProperties:cookieProperties];
[[NSHTTPCookieStorage sharedHTTPCookieStorage] setCookie:cookie];

添加Header

AFHTTPRequestSerializer *serializer = [AFHTTPRequestSerializer serializer];
        
NSDictionary* dic = @{@"a":@"a",@"b":@"b",@"c":@"c",@"d":@"d"};
        NSString* json = [Util convertObjectToJSON:dic];
json = [json stringByReplacingOccurrencesOfString:@"\n" withString:@""];//有换行，没法添加，要去掉换行
[serializer setValue:json forHTTPHeaderField:@"Security-Token"];
[manager setRequestSerializer:serializer];

POST修改content type为application json

NSMutableURLRequest *request=[[NSMutableURLRequest alloc]initWithURL:[NSURL URLWithString:url]
                                                                 cachePolicy:NSURLRequestReloadIgnoringCacheData
                                                             timeoutInterval:kTimeOut];
[request setHTTPMethod: @"POST"];
[request setValue: @"application/json" forHTTPHeaderField:@"Content-Type"];
[request setHTTPBody:[[Util convertObjectToJSON:parameters] dataUsingEncoding:NSUTF8StringEncoding]];
        
AFHTTPRequestOperation *currentoperation = [manager HTTPRequestOperationWithRequest:request success:^(AFHTTPRequestOperation * _Nonnull operation, id  _Nonnull responseObject) {
            [UIApplication sharedApplication].networkActivityIndicatorVisible = NO;

            [UIApplication sharedApplication].networkActivityIndicatorVisible = NO;
            if(error.code == -999){}//取消请求，不用返回
        }];
[currentoperation start];

xcode使用workspace管理项目

1.创建一个工作空间的文件夹
2.xcode新建一个workspace
3.新建项目，或者把原始项目copy到该目录下面
4.创建公共静态库项目
5.自己的项目中引用自己的静态库。
添加Header Serach Paths,User Header Serach Paths,并且类型为recursive不是non-recursive

引用之前已经添加过pod的项目，会出现diff: /../Podfile.lock: No such file or directory错误信息
把该项目的pods目录下的xcconfig文件删除，从新install下pod就可以了
继续报错：ld: library not found for -lPods
删除build phases-link binary with libraries-libPods.a

去掉后台持续定位的蓝条

NSLocationWhenInUseUsageDescription 允许在前台使用时获取GPS的描述
NSLocationAlwaysUsageDescription 允许永远可获取GPS的描述
这两个不要一块添加，如果两个都添加，默认采用NSLocationWhenInUseUsageDescription。
永远后台的时候只需要设置allowsBackgroundLocationUpdates为yes后就没有蓝条了
