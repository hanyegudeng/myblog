使用脚本自动打包并发布APP到fir.im

2016/8/9 posted in  iOS

记录一下操作步骤吧
1.clean一下对应的target
xcodebuild -target TARGETNAME clean

2.用xcode编译生成archive文件
xcodebuild -workspace XXX.xcworkspace -scheme XXX CODE_SIGN_IDENTITY='iPhone Distribution：xxx' -archivePath "~/ipa/2016-08-09" archive

3.导出ipa文件
这里特备注意，使用xcodebuild没找到如果把证书导进去，一直没法安装，最后使用了xcrun命令
xcrun -sdk iphoneos -v PackageApplication xxx.app -o xxx.ipa

4.上传ipa到fir.im上
fir publish 'xx.ipa' -T 自己fir上的token

纠结的一点，在CODE_SIGN_IDENTITY，不知道后面的证书描述怎么获取的。找了半天，后来才返现，是在keychain里面的证书的常用名称字段直接复制过来就可以。

下面附上最终的脚本


echo "请选择要发布的版本："
echo "1.测试版本"
echo "2.生成版本"

read -p "输入编译的版本序号: "  number

echo $number

case "$number" in
1 ) projectName="xxx" ;;
2 ) projectName="xxxPro" ;;
* ) echo "输入错误!";exit 0 ;;
esac

echo "Publish..............................."
day=`date "+%Y-%m-%d"`
currenttime=`date "+%H-%M"`

mkdir -p $HOME/ipa/$day/$currenttime

echo "清空项目，重新编译"
xcodebuild -target $projectName clean

echo "开始编译，导出archive"
xcodebuild  -workspace xxx.xcworkspace -scheme $projectName CODE_SIGN_IDENTITY="iPhone Distribution: xxx" -archivePath $HOME/ipa/$day/$currenttime/$projectName archive

echo "导出ipa文件"
xcrun -sdk iphoneos -v PackageApplication $HOME/ipa/$day/$currenttime/$projectName.xcarchive/Products/Applications/$projectName.app -o $HOME/ipa/$day/$currenttime/$projectName.ipa

echo "上传到FIR"
fir publish $HOME/ipa/$day/$currenttime/$projectName.ipa -T token
