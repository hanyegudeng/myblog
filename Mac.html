Git统计每个人提交的Commit次数和代码行数

提交次数git shortlog --numbered --summary
查看所有的commit数git log --oneline | wc -l
提交删除行数git log --author="$(git config --get user.name)" --pretty=tformat: --numstat | awk '{ add += $1 ; subs += $2 ; loc += $1 - $2 } END { printf "added lines: %s removed lines : %s total lines: %s\n",add,subs,loc }'
使用cloc统计代码行数给力：
安装brew install cloc
统计目录下的所有文件，代码行数cloc ./
排除文件夹cloc ./ --exclude-dir=Pods,Tests,Utility

Ubuntu+Nginx+Django+Uwsgi配置

1.安装pip：apt-get install python-pip
2.安装django：pip install django
3.安装nginx：apt-get install python-dev nginx
4.安装uwsgi：pip install uwsgi
5.创建django项目

cd /var/
mkdir html
cd html
django-admin startproject p2p
cd p2p
vi p2p.xml
<uwsgi>
    <socket>:8000</socket>#端口号
    <chdir>/var/html/p2p/p2p</chdir>#项目路径
    <module>wsgi</module> 
    <daemonize>uwsgi.log</daemonize>#运行日志
    <pythonpath>/var/html/p2p/</pythonpath>#对应wsgi.py路径，这个错了会导致后面运行报错。这个地址不是python的路径
</uwsgi>

6.修改nginx配置
位置在：/etc/nginx/sites-available
该配置文件需要自己先添加，然后ln到sites-enabled里面

server {
    listen 80;#web服务器端口号
    root /var/html/p2p/;#项目路径
    index index.html index.htm;
    server_name xxx.com;
    location / {
        include uwsgi_params;
      uwsgi_pass 127.0.0.1:8000;#uwsgi对应端口号
    }
}

7.重启nginx/etc/init.d/nginx restart
8.运行uwsgiuwsgi /var/html/p2p/p2p.xml
9.查看运行日志
10.访问静态文件

配置nginx
location ~ ^/static/ {
            root    /var/html/p2p;
        }

下面是一些调试时候用到的命令：
ps -ef | grep uwsgi
killall -9 uwsgi
netstat -nap | grep 8000查看端口
cat /etc/services | grep 8000

开机启动uwsgi
编辑/etc/rc.local这个文件，在exit 0前加入如下 uwsgi /var/html/p2p/p2p.xml

Xcode6插件管理器失效的解决办法

1.打开终端，输入以下代码获取到DVTPlugInCompatibilityUUID
defaults read /Applications/Xcode.app/Contents/Info DVTPlugInCompatibilityUUID
2.然后输入如下命令 【最后一项是获取到的DVTPlugInCompatibilityUUID】
find ~/Library/Application\ Support/Developer/Shared/Xcode/Plug-ins -name Info.plist -maxdepth 3 | xargs -I{} defaults write {} DVTPlugInCompatibilityUUIDs -array-add 9F75337B-21B4-4ADC-B558-F9CADF7073A7
3.重启Xcode。然后shift+command+9就可以调出插件管理器了

Ubuntu12.04搭建FTP服务器

  • 安装好系统后，安装vsftp服务器 apt-get install vsftpd
  • 安装好以后默认是匿名登录。配置文件在/etc/vsftpd.conf这里，修改之
listen=YES
anonymous_enable=NO#关闭匿名登录
local_enable=YES
write_enable=YES
pam_service_name=vsftpd
rsa_cert_file=/etc/ssl/private/vsftpd.pem
local_root=/var/html#修改ftp访问路径
userlist_enable=YES
userlist_deny=NO
userlist_file=/etc/vsftpd.user_list#ftp用户列表
  • 添加ftp用户
useradd -d /var/ftp -s /usr/sbin/nologin ftptest
  • 修改ftp用户密码
passwd ftptest
  • 编辑用户列表
ftptest
  • 修改ftp目录路径
chown ftptest /etc/html
  • 删除默认用户配置（这个困扰很久，不删除没法登录）
vi /etc/pam.d/vsftpd

#auth   required        pam_listfile.so item=user sense=deny file=/etc/ftpusers onerr=succeed 注释掉这一行

# Note: vsftpd handles anonymous logins on its own. Do not enable pam_ftp.so.

# Standard pam includes
@include common-account
@include common-session
@include common-auth
#auth   required        pam_shells.so  注释掉这一行
  • 到此结束，上传下载搞定
