如何是用Targets来管理开发测试包

2016/6/28 posted in  iOS

我们在开发APP的过程中难免会遇到开发测试分离的情况，最常见的就是API地址，是不同的，这个可以通过宏定义或者注释之类的来加以解决，但是打包的时候万一忘了就完蛋喽。其实xcode就能帮我们防呆，那就是创建多个Target来管理不同的环境，配置不同的plist以及对应的程序图标版本号等等的信息。具体做法如下：

1.选择原有的target，复制一份，名字可以修改，也可以不管
2.在Build Settings里面的Preprocessor Macros添加不同的DEVELOPMENT=0
3.在用到API host的地方加入判断

4.如果是用swift编写项目，需要在Swift Compiler - Custom Flags里面把DEVELOPMENT添加进去才可以起效
```

if DEVELOPMENT

define SERVER_URL @"http://dev.server.com/api/"

define API_TOKEN @"DI2023409jf90ew"

else

define SERVER_URL @"http://prod.server.com/api/"

define API_TOKEN @"71a629j0f090232"

endif


4.如果程序中使用cocoapods，并且使用了workspace，就要稍微配置下podfile文件了。

abstract_target 'targetBlock' do

platform :ios, "7.0"

xcodeproj 'xx/xx.xcodeproj'

pod 'AFNetworking', '~> 3.0'

target 'testing target'
target 'product target'

end
```

5.如果后续添加文件什么的，记得选择两个target哦。要不另外一个target就没有对应的文件喽

6.注意：
swift需要在Other Swift Flags里面添加
"-D" "DEVELOPMENT"
